# Custom String Implementation (SSO & FBString Optimization)

Этот проект представляет собой реализацию шаблонного класса `BasicString`, во многом следующую интерфейсу `std::string`, с поддержкой Small String Optimization

## 🎯 Цели проекта

* **Изучение Small String Optimization (SSO):** Реализация механизма, позволяющего хранить короткие строки без выделения памяти в куче.

## 🚀 Ключевые особенности

### 1. FBString SSO Layout
Данный проект использует схему **FBString** (Facebook String). Внутри класса используется `union`, который переключается между двумя режимами:

* **Short Mode (Stack):** Позволяет хранить до 23 символов (для `char`) прямо внутри объекта.
* **Long Mode (Heap):** Хранит указатель на динамическую память, capacity и size.

#### Схема распределения памяти:

| Режим | Данные | Флаг / Размер |
| :--- | :--- | :--- |
| **Short** | Буфер 23 байта | 7 бит под размер, 1 бит — флаг типа |
| **Long** | Указатель (8B) + Capacity (8B) | 7 бит под размер (LSB), 1 бит — флаг типа |

### 2. Система CharTraits
Логика работы с символами (копирование, сравнение, поиск длины) вынесена в отдельный шаблон `CharTraits`.
* **Базовый шаблон:** Универсальная реализация для любых типов символов.
* **Специализация для `char`:** Использует функции стандартной библиотеки C (`std::memcmp`, `std::memcpy`, `std::strlen`), что значительно ускоряет обработку char-строк.

### 3. Функционал
Реализован набор стандартных методов:
* **Модификаторы:** `push_back`, `pop_back`, `append`, `clear`, `resize`, `reserve`, `shrink_to_fit`.
* **Доступ:** `operator[]`, `at`, `front`, `back`, `data`.
* **Операторы:** Конкатенация (`+`, `+=`), сравнение (`==`, `<=>`).
* **User-defined Literal:** Поддержка литерала `""_s` для быстрого создания объектов `String`.
